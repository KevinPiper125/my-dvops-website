Cordino Signup App

A full-stack Android app with a RESTful Laravel backend — built end to end.

← Back to Home

Overview

What I Built

The Cordino Signup App is a native Android application built with Kotlin in Android Studio. It provides a clean, functional user signup flow — collecting user information, validating input on the client side, and communicating securely with a backend API to register and manage users.

What made this project a full-stack challenge was building and managing the backend myself using Laravel — a PHP framework for building RESTful APIs. That meant designing the API endpoints, handling authentication logic, and making sure the Android app and server could talk to each other reliably.

How It Works

Architecture

The app follows a straightforward client-server model. The Android front end handles the UI — forms, validation, and user feedback. When a user submits the signup form, the app sends an HTTP request to the Laravel REST API, which processes the data, stores it, and returns a response.

Working with RESTful services meant learning how HTTP methods work in practice — POST for creating users, GET for retrieving data — and how to parse JSON responses in Kotlin to update the UI dynamically.

Kotlin Android Studio Laravel (PHP) REST API JSON HTTP / Retrofit

What I Learned

Key Takeaways

This project pushed me into full-stack territory for the first time. Building the API with Laravel and consuming it in Kotlin forced me to think about how data flows between a mobile client and a server — and all the places things can go wrong in between.

It also reinforced good habits around input validation, error handling, and keeping concerns separated between the front end and backend.
